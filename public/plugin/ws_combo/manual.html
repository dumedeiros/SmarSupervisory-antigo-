ws:combo user's manual

ws:combo is a replacement for a standard <select /> tag that extends the dropdown control with additional features, such as images, custom CSS styling, and more. It is built on top of popular JavaScript framework jQuery and it is easy to implement and customise.

The main aim of this control is to replace the standard <select /> input control due to its lack of styling support. In addition to visual enhancements, ws:combo offers intuitive keyboard control using arrow keys, pg up, pg down and other.

1. Requirements

Following requirements must be met to successfully use this control:

  • A browser with JavaScript support enabled. Following browsers have been tested and are fully supported:
    • Chrome,
    • Firefox,
    • IE 7+,
    • Opera and Safari.
    Note: ws:combo has also been successfully tested with IE6 but it is not recommended to use it with this browser as not all features are supported.
  • A jQuery framework version 1.6 or newer - this can be downloaded from here: http://docs.jquery.com/Downloading_jQuery.
  • Good knowledge of JavaScript and JSON.
    Note: The examples below use JSON syntax.

2. Installation

The package contains all necessary files in the ws/ folder. Include the ws/ws-combo.js script in your document (make sure to adjust the path to the script). The jQuery library needs to be included in the document before the ws-combo.js script:

<script type="text/javascript" src="http://ajax.googleapis.com/ajax/libs/jquery/1.7.1/jquery.min.js"></script>
<script type="text/javascript" src="ws/ws-combo.js"></script>

The ws-combo.js javascript file will automatically add the required stylesheet to the document. For less data transfer you can use the minified version of the script (~ 40% less size) which is available in ws/ws-combo.min.js.

Note: To properly load the styles, the ws-combo.css and ws-combo.gif files must be placed in the same directory as ws-combo.js!

3. Examples

3.1 The following code demonstrates how to create a simple standalone dropdown:

<script type="text/javascript">
$.wscombo({
	id:'comboId1',  // unique combo id
	name:'combo1',  // form element name
	items:[         // items
		{t:'United States',v:'US'},
		{t:'United Kingdom',v:'GB'},
		'-',                      // separator
		{t:'Albania'},
		{t:'Alpha Centauri',d:1}, // disabled item
		{t:'Czech Republic'},
		{t:'Russia'}
	]
});
</script>

3.3 Example of a standalone combo with more features including images and HTML code in the items:

<script type="text/javascript">
$.wscombo({
	id:'comboId3',  // unique combo id
	name:'combo3',  // form element name
	img:'http://ws-cdn.appspot.com/lib/countries.gif', // path to main image file - if you do not want to use imagemap
	                                                   // set this to 1 (or true) to enable images
	reset:'Clear',  // allow user to reset the control to an empty state plus specify custom reset title
	combo:'or enter a new country:', // allow user to add custom values plus specify custom combo title
	text:'(select country)', // default text
	width:110,      // fixed width of the control input field
	maxHeight:280,	// max height of scrollable area with items
	maxLength:20,   // max 20 characters for the custom combo text
	validate:/^[^0-9]+$/, // combo validation regexp sample - combo textbox cannot contain numbers
	onselect:function(i){$('#spComboId3').text(i?'You have selected '+(i.t||i.combo):'');}, // onselect handler
	items:[         // items
		{l:'Topmost Countries'}, // label
		{
			t:'United States', // text (also used as a display value)
			v:'US',            // field value (if omitted then text will be used)
			i:{x:0,y:-3760},   // offset of the icon within the main image file
			// html code to be used instead of the text displayed
			h:'<b style="display:inline-block;margin-top:5px;line-height:120%">United States<br />'
			 +'<i style="font-weight:normal;font-size:10px;color:#999">(including minor outlying islands)</i></b>'
		},
		{
			h:'<b>United Kingdom</b>', // html code - note that we have omitted text property,
									   // this will be extracted from the html code !
			v:'GB',
			i:{x:0,y:-3744}
		},
		{l:'Other Countries'}, // label
		{
			v:'AF',
			t:'Afghanistan',
			// if you want to use single image then specify the url
			i:'http://ws-cdn.appspot.com/lib/countries.gif'
		},
		{v:'AL',t:'Albania',i:{x:0,y:-32}}
		/*...*/
	]
});
</script>

4. Documentation

The wscombo() method accepts a JSON object with following parameters:

Parameter Type Description
combo Bool (or Number) Default false (or 0). True (or 1) to allow user to enter custom text; false (or 0) otherwise. If the combo option is allowed, the default text is 'Enter a new value:'. See below how to override this.
String If the combo parameter is string, it is used as a title for the custom combo text option. In this case the combo option is always present.
id String Default null. Unique combo identifier. If this parameter is null or not provided, a random id will be used instead. If you provide this parameter, you can later access the combo instance using Ws.Combo.your_combo_id.
img Bool (or Number) Default false (or 0). True (or 1) to display the image box and image icons in the items; false otherwise.
String Path to main image file. Once the path has been set, user can use xy-offset within the main image file for the single combo items.
items * Array Array of the combo items. An item can be a dash character '-' (specifies a separator) or a JSON object with following parameters:
  • disabled or d - specifies that the item is disabled and cannot be selected.
  • html or h - html code to be displayed instead of the text.
  • label or l - label title; suitable for grouping the items. Label is unselectable.
  • separator or s - if set to true, item acts as a separator. Separator is unselectable.
  • text or t - text (if html not used); this is also used for displaying the currently selected item. If not provided then text is extracted from the html option above.
  • value or v - value returned for the form element. If not used the text is returned instead.
* this options is required for a standalone combo!
maxHeight Number Default 200. Maximum height in pixels the items area can take. If the items area is over this height, the scrollbar is added.
maxLength Number Default 0 (unlimited). The maximum number of characters of the custom combo text. If zero is specified the number is unlimited.
name String Optional form element name.
reset Bool (or Number) Default false (or 0). True (or 1) to allow user to reset the combo; false (or 0) otherwise. If the reset option is turned on, the default text is 'Reset'. See below how to override this.
String If the reset parameter is string, it is used as a title for the custom reset option. In this case the reset option is always present.
selectedIndex Number Specifies the initial selected item index. If this parameter is not specified the value is used to determine the selectedIndex (see below).
text String Default '(select)'. The placeholder text. Used if selectedIndex=-1.
validate Function User defined validation function for the custom combo text. This function returns a boolean value. True if the text is valid; false otherwise.
RegExp A regular expression object used to validate the custom combo text.
value Object The selected value.
width Number Default 'auto'. The width, in pixels, of the inner textfield area that shows the currently selected item. If set to auto, the width is calculated automatically based on the items content.
Handlers
onselect(e) Occurs on several circumstances:
  • when an item is selected (parameter e is the item),
  • or the combo is reset (parameter is empty),
  • or a custom text has been selected (e.combo is the text).

You can access the combo instance using Ws.Combo.your_combo_id. See example 3.5 for more details. The following methods are available to manipulate the control:

Method Returns Description
select(index[,keepOpen]) object Manually select an item.
  • index - zero based index of the item to select
  • keepOpen - optional boolean parameter; if set to true the popupbox will stay opened; otherwise it will be hidden.
Returns the self-reference (current Ws.Combo instance).
reset([keepOpen]) object Manualy reset the combo.
  • keepOpen - optional boolean parameter; if set to true the popupbox will stay opened; otherwise it will be hidden.
Returns the self-reference.
enable() object Enable the control. Returns the self-reference.
disable() object Disable the control. Returns the self-reference.
show() object Show the popupbox. Returns the self-reference.
hide() object Hide the popupbox. Returns the self-reference.
set(params) object Set's the combobox parameters. This method is commonly used to set the items of the control after the combo has been created.
  • params - dictionary of combo parameters - these use the same format as parameters used to create the combo - see parameters documentation.
Returns the self-reference.
insert([index,]item1,item2,...)
insert([index,]items) 		object Insert items to the combo.
  • index - index to insert the item(s) at. If not provided the items will be appended.
  • item1, item2, ... or items (Array) - the items to insert. The format of the item is described here.
Returns the self-reference.
remove(index[,count]) object Removes items from the combo.
  • index - the index to start removing at.
  • count - The number of items to remove. If not set then this is equal to 1.
Returns the self-reference.

5. Styling & Themes

ws:combo comes with predefined default styles in ws/ws-combo.css. If you want to use your own styles, simply edit the stylesheet file and change the appropriate styles. Note that intermediate knowledge of CSS is required for this. There are no additional themes available with the release of the current version. This is planned with the release of the next major version.

You can also override the default styles by adding a class to a parent container and creating the styles for that class:

/* Styles for the following combo declaration: <div class="customCombo1"><select id="comboId5">...</select></div> */
#comboId5.dropdown-txt{/* text field font */
	font-family: "Trebuchet MS", "Helvetica Neue", Arial, sans-serif;
	font-size: 13px;
	color: #476896
}
#comboId5.combo-sel{color:#758eb0 !important;opacity:.5 !important}
#comboId5::selection{color:#476896 !important}
#comboId5::-moz-selection{color:#476896 !important}
.customCombo1 div.ws-combo-items li{/*item*/font-family:"Trebuchet MS",sans-serif;font-size:13px;color:#476896}
.customCombo1 div.ws-combo-items li.ws-combo-in{/*selected item*/background-color:#476896;color:#fff}
.customCombo1 div.ws-combo-items li.ws-combo-separator s{/*separator*/background-color:#476896}
.customCombo1 div.ws-dropdown{/*borders-background*/border:1px solid #476896;background:#fdfbfa}
.customCombo1 div.ws-dropdown-above div.ws-dropdown-popup{border:1px solid #476896;border-bottom:0}
.customCombo1 div.ws-dropdown-below div.ws-dropdown-popup{border:1px solid #476896;border-top:0}
.customCombo1 div.ws-dropdown-below .dropdown-btn,
.customCombo1 div.ws-dropdown-above .dropdown-btn{border-left:1px solid #476896}
.customCombo1 div.ws-dropdown-popup{background-color:#476896}
.customCombo1 div.ws-combo-footer{/*combo area*/background:#dae1ea;font-family:"Trebuchet MS",sans-serif;color:#476896}
.customCombo1 div.ws-combo-footer input{/*combo textfield*/font-family:"Trebuchet MS",sans-serif}

6. Known Issues and Limitations

01 Container Overflow Hidden - if the combo is placed within a DIV container with fixed height and overflow hidden, the popup items might not be visible. The workaround is to place the combo outside the container.

02 Z-Index - if, for some reason, the combo/popup div is not visible, try adjusting the z-indexes in the stylesheet.

03 Shadows - combo shadows are not available on older browsers that do not support css 3.

04 CSS Hacks - ws-combo.css will not validate as it contains browser specific hacks to make this component work on older browsers. A plan to fix this is in the roadmap for the next major version.

05 Touch Support - support for touch enabled devices is not fully implemented.
